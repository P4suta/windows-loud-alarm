# Build perf baseline — 2026-05-27

Initial baseline through the full "Data-Driven Local Build Optimization" change set,
captured in three measurement passes:

1. **Initial** — repo as it was before any change in this session.
2. **After Phase 2** (label: `phase-4-before`) — packages.lock.json, xUnit parallel,
   UseArtifactsOutput, RIDs everywhere, ReportAnalyzer, Task.Delay → Yield/StateStream.
3. **After Phase 4** (label: `phase-4-after`) — AlarmStore `Pending`/`IsIdle`/`WaitForIdleAsync`,
   replacing the last `Task.Yield()` loop in tests; MeasureBuild.cs labeled output;
   analyzer parser fixed (now extracts the csc.exe table via `-tl:off -clp:Verbosity=detailed`).

Per-Phase results are committed alongside as
`baseline-2026-05-27-phase-4-before.md` and `baseline-2026-05-27-phase-4-after.md`
so each step can be replayed.

Each pass is the median of 3 `dotnet build`/`dotnet test` runs.

## What changed (chronological)

### Phase 1 — measurement infrastructure
1. `scripts/MeasureBuild.cs` (.NET 10 file-based program) + `just measure`.
2. `just measure-binlog` / `just analyzer-report` / `just lint-build`.
3. `docs/perf/README.md` describing the procedure.

### Phase 2 — high-confidence cleanups (no measurement required)
4. `packages.lock.json` + `dotnet restore --locked-mode` for reproducibility.
5. Windows RIDs on every project so the lock file stays stable across publish.
6. `tests/*/xunit.runner.json` enables `parallelizeAssembly` + `maxParallelThreads=-1`.
7. R3-driven `WaitForStateAsync` (replaces `Task.Delay(10)` polling) — biggest single win.
8. `<UseArtifactsOutput>true</UseArtifactsOutput>` collapses every `bin/`/`obj/` into `artifacts/`.
9. `<ReportAnalyzer Condition="Debug">true</ReportAnalyzer>` for ongoing analyzer visibility.
10. `lefthook.yml` pre-push runs `just test` (CI doesn't exist yet, so this is the line of defence).
11. `.editorconfig` relaxes `CA1303` in `scripts/`, downgrades `BC0301` to warning.

### Phase 4 — finishing the previously-deferred residue
12. `AlarmStore.Pending` / `IsIdle` + `Harness.WaitForIdleAsync()` in tests. Removes the
    last fixed wall-clock budget (`Task.Yield()` loop). Counter is `Interlocked` and the
    BehaviorSubject is OnNext'd on every dispatch / reducer completion, so the test
    deterministically waits for an empty queue.
13. `MeasureBuild.cs` writes `baseline-<date>-<slug>.md` when `--label` is set, so per-Phase
    runs can coexist (no overwrite). Cold-rebuild is now measured first because its
    `just clean` would otherwise wipe `artifacts/perf` between captures.
14. `just analyzer-report` adds `-tl:off -clp:Verbosity=detailed` — the terminal logger
    and normal verbosity were swallowing csc.exe's analyzer summary. MeasureBuild.cs now
    parses both Japanese ("時間 (秒)") and English ("Time (s)") locales.
15. `.editorconfig` adds `MA0009 / MA0110 / SYSLIB1045 / CA1308 = none` for `scripts/` —
    source-generated regex isn't available in a file-based program, and ToLowerInvariant
    is correct for ASCII slug filenames.

## Workflow for future optimizations

```pwsh
just measure --iterations 5 --label "before <change>"   # commit the resulting md
# … make the change …
just check
just measure --iterations 5 --label "after <change>"    # commit the resulting md
```

`--label` produces `baseline-<date>-<slug>.md` so the "before" run isn't overwritten.

## Caveats

- 3 iterations exposes order-of-magnitude regressions but not ±5 % swings. For decisions on
  marginal changes, run `just measure --iterations 7` or more.
- The Downloads-folder repo location triggers MSBuild BuildCheck `BC0301`. We downgrade it
  to warning rather than move the repo.
- After-Phase-4 `cold-rebuild` had a 28.6 s outlier on iteration 1 (median 17.6 s is the
  honest number). Likely background scanner interference, not a regression.
- **Phase 2 sub-step deltas (2.1 lock file / 2.2 xUnit parallel / 2.3 Task.Delay→stream /
  2.4 UseArtifactsOutput / 2.5 ReportAnalyzer) are NOT captured individually.** Only the
  bulk Initial→Phase-2 delta exists. The `--label` workflow now exists to do per-step
  before/after, but reconstructing the Phase 2 splits would require reverting each change
  in sequence — there is no git in this repo right now. Going forward, every new
  optimization should produce a labeled before/after pair on the spot.
